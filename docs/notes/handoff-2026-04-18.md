## Public API (sketch)

```rust
use lensfun::{Database, Modifier};

// Load. Bundled has no I/O; load_dir reads from disk.
let db = Database::load_bundled()?;
// or: let db = Database::load_dir("/usr/share/lensfun/")?;

// Find a lens for a body + lens-name string.
let cam = &db.find_cameras(Some("Canon"), "EOS R5")[0];
let lens = &db.find_lenses(Some(cam), "Canon EF 24-70mm f/2.8L II USM")[0];

// Build a per-shot modifier. `crop` is the *body's* crop factor.
let mut m = Modifier::new(lens, /* focal */ 35.0, /* crop */ 1.0,
                          /* w */ 6720, /* h */ 4480, /* reverse */ true);
m.enable_distortion_correction(lens);
m.enable_tca_correction(lens);
m.enable_vignetting_correction(lens, /* aperture */ 4.0, /* distance */ 5.0);

// Apply to a coordinate buffer (one row at a time).
let mut coords = vec![0.0_f32; 6720 * 2];
m.apply_geometry_distortion(0.0, /* y */ 100.0, 6720, 1, &mut coords);

// Apply to a pixel buffer (vignetting).
let mut pixels = vec![0.0_f32; 6720 * 4480 * 3];
m.apply_color_modification_f32(&mut pixels, 0.0, 0.0, 6720, 4480, 3);
```

## Known divergences from upstream

Each is intentional and documented at the call site. None are bugs.

1. **ACM models**: parsed by `db.rs` and stored as `None`. See `db.rs` TODOs.
2. **Crop-factor loop in `MatchScore`** iterates over the lens's single `crop_factor` rather than upstream's `lfLensCalibrationSet` vector — Rust port has flat calibration Vecs by design.
3. **`find_cameras` / `find_lenses` signatures** are simplified relative to upstream (no `sflags`, no `maker` arg on `find_lenses`). Implementation matches upstream defaults.
4. **`fuzzy_str_cmp` casefolding** uses Rust `str::to_lowercase` instead of glib's case-fold tables. Functionally equivalent for ASCII, which is all the upstream test suite exercises.
5. **u16/u8 vignetting** uses upstream's 20.12 / 22.10 fixed-point arithmetic + `clampbits` (not naive `(v + 0.5) as i32`) — needed for bit-exact match with regression test.

## Risks for future sessions

1. **Float bit-exactness** continues to matter. Don't refactor accumulation order.
2. **Multi-language strings**: parsed but not surfaced to query results yet. When `Camera::model_localized` etc. matter, wire them into `MatchScore` (it currently only matches against the default name).
3. **No end-to-end image test exists yet.** All current tests are scalar/buffer-level. Add at least one test that runs a small synthetic image through the full pipeline once you have a real test image.
